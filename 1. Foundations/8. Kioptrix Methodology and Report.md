🧠 KIOPTRIX METHODOLOGY

==================================================



📌 Overview

\--------------------------------------------------



Kioptrix is a well-known vulnerable virtual machine used for penetration testing practice.



It is commonly used to learn the methodology of attacking a system from initial discovery to full compromise.



The goal is to simulate the steps a penetration tester follows during an engagement.



\--------------------------------------------------

⚔️ Standard Pentesting Methodology

\--------------------------------------------------



The process typically follows these stages:



1️⃣ Reconnaissance  

2️⃣ Enumeration  

3️⃣ Exploitation  

4️⃣ Privilege Escalation  

5️⃣ Maintaining Access  

6️⃣ Documentation and Reporting



\--------------------------------------------------

🔎 Phase 1 — Reconnaissance

\--------------------------------------------------



The first step is identifying hosts on the network.



Common tools:



nmap  

netdiscover  

arp-scan  



Example:



nmap -sn 192.168.1.0/24



Goal:



Identify live systems on the network.



\--------------------------------------------------

📡 Phase 2 — Enumeration

\--------------------------------------------------



Once a target is discovered, deeper information gathering begins.



Examples include:



• open ports

• running services

• software versions

• potential vulnerabilities



Example:



nmap -sS -sV -T4 192.168.1.102



This identifies:



• open ports

• service versions

• possible attack vectors



\--------------------------------------------------

💥 Phase 3 — Exploitation

\--------------------------------------------------



After identifying vulnerabilities, exploitation attempts begin.



This may include:



• known exploits

• misconfigurations

• vulnerable services



Example tools:



Metasploit  

Searchsploit  

Custom scripts



\--------------------------------------------------

🔑 Phase 4 — Privilege Escalation

\--------------------------------------------------



After initial access is obtained, the attacker attempts to gain higher privileges.



Examples:



• local exploits

• weak permissions

• misconfigured services



Goal:



Gain root or administrator access.



\--------------------------------------------------

📂 Phase 5 — Post Exploitation

\--------------------------------------------------



After full access is achieved, additional information may be gathered.



Examples:



• password hashes

• system configuration

• sensitive data



\--------------------------------------------------

📝 Phase 6 — Reporting

\--------------------------------------------------



The final stage of penetration testing is documentation.



A report includes:



• discovered vulnerabilities

• proof of exploitation

• risk impact

• remediation recommendations



\--------------------------------------------------

🛡️ Security Relevance

\--------------------------------------------------



Learning methodology is just as important as learning tools.



Penetration testing is not about randomly running tools — it is about following a structured process to identify and exploit vulnerabilities.

📄 KIOPTRIX LAB REPORT

==================================================



📌 Overview

\--------------------------------------------------



This lab demonstrates a full penetration testing workflow against a vulnerable Kioptrix virtual machine.



The objective is to discover the system, identify vulnerabilities, exploit them, and gain root access.



\--------------------------------------------------

🎯 Target Information

\--------------------------------------------------



Target VM: Kioptrix  

Attacker Machine: Kali Linux  

Network Type: Virtual Lab Environment



\--------------------------------------------------

🔎 Step 1 — Network Discovery

\--------------------------------------------------



First, identify live hosts on the network.



Command:



netdiscover



Example output:



192.168.1.102  08:00:27:aa:bb:cc  Kioptrix VM



The target machine has been identified.



\--------------------------------------------------

📡 Step 2 — Port Scanning

\--------------------------------------------------



Next, scan the target system to identify open ports.



Command:



nmap -sS -sV -T4 192.168.1.102



Example results:



22/tcp   open  ssh

80/tcp   open  http

139/tcp  open  netbios

445/tcp  open  smb



This reveals several potential attack vectors.



\--------------------------------------------------

🔎 Step 3 — Service Enumeration

\--------------------------------------------------



Further investigation focuses on the discovered services.



Example tools:



nikto  

enum4linux  

dirb  



These tools help identify:



• hidden directories

• SMB information

• web server vulnerabilities



\--------------------------------------------------

💥 Step 4 — Exploitation

\--------------------------------------------------



Once a vulnerability is identified, exploitation is attempted.



Example:



searchsploit apache



A known Apache vulnerability may be used to gain shell access.



\--------------------------------------------------

🔑 Step 5 — Privilege Escalation

\--------------------------------------------------



After obtaining shell access, the next objective is gaining root privileges.



Common checks include:



sudo -l  

find / -perm -4000 2>/dev/null



These commands identify possible privilege escalation paths.



\--------------------------------------------------

🏆 Final Result

\--------------------------------------------------



Root access successfully obtained.



Proof:



id



Output:



uid=0(root) gid=0(root)



\--------------------------------------------------

📝 Key Lessons

\--------------------------------------------------



This lab demonstrates the importance of:



• systematic enumeration

• identifying vulnerable services

• understanding Linux privilege escalation

• documenting each step of the process



\--------------------------------------------------

🛡️ Security Impact

\--------------------------------------------------



If this system were in a real environment, the vulnerabilities discovered could allow attackers to gain full control of the server.



This highlights the importance of:



• patch management

• secure configurations

• continuous security monitoring
