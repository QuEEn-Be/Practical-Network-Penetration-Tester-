📘 MAN PAGES (MANUAL PAGES)
==================================================

📌 Overview
--------------------------------------------------

The "man" command stands for manual.

Man pages provide built-in documentation for commands installed on a Linux system.

They are one of the most important tools for learning how commands work, understanding their options, and troubleshooting issues.

Almost every Linux command includes a manual page that explains:

• command syntax
• available options
• usage examples
• command descriptions

Learning how to read man pages is an essential skill for both system administrators and penetration testers.

--------------------------------------------------
📖 Accessing a Man Page
--------------------------------------------------

Command:

man command_name

Example:

man ls

This opens the manual page for the "ls" command.

The manual page will contain information such as:

NAME
Description of the command

SYNOPSIS
Command syntax and arguments

DESCRIPTION
Detailed explanation of the command

OPTIONS
Available command options and flags

EXAMPLES
Practical examples of command usage

--------------------------------------------------
🧭 Navigating a Man Page
--------------------------------------------------

Man pages are opened using the "less" pager, which allows navigation within the document.

Common navigation keys include:

SPACE
Move forward one page

b
Move backward one page

ENTER
Move forward one line

/word
Search for a word within the document

n
Move to the next search result

q
Quit the man page

--------------------------------------------------
🔎 Searching for Commands with apropos
--------------------------------------------------

If you do not know the exact command name, you can search the manual database.

Command:

apropos keyword

Example:

apropos password

This searches for commands related to the keyword "password".

Example output might include:

passwd (1)           - change user password
chpasswd (8)         - update passwords in batch mode

--------------------------------------------------
📚 Viewing a Short Command Description
--------------------------------------------------

Another useful command is:

whatis

Example:

whatis ls

Output:

ls (1) - list directory contents

This command gives a short description of a command from the man page database.

--------------------------------------------------
📌 Manual Page Sections
--------------------------------------------------

Linux manual pages are divided into numbered sections.

Common sections include:

1
User commands

2
System calls

3
Library functions

4
Special files and drivers

5
File formats and configuration files

6
Games and entertainment

7
Miscellaneous information

8
System administration commands

Example:

man 5 passwd

This displays the manual page for the passwd file format.

--------------------------------------------------
📚 Key Takeaways
--------------------------------------------------

✔ Man pages provide detailed documentation for Linux commands.

✔ The command "man command" displays the full manual page.

✔ Navigation within man pages is similar to the "less" command.

✔ The "apropos" command helps find commands related to a keyword.

✔ The "whatis" command shows a short description of a command.

--------------------------------------------------
🛡️ Security Relevance
--------------------------------------------------

Security professionals frequently use man pages when working with unfamiliar tools or commands.

Because penetration testing environments contain many specialized utilities, the ability to quickly reference documentation directly from the system is extremely valuable.

Man pages allow practitioners to learn new commands without needing external documentation.
