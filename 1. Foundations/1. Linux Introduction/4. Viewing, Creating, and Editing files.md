=============================================

 VIEWING, CREATING, AND EDITING FILES

=============================================


📌 Overview

\--------------------------------------------------



Linux provides powerful command-line tools for creating, viewing, and modifying files.



During system administration and penetration testing, these commands are used constantly to:



• create scripts

• inspect configuration files

• analyze logs

• store command output

• manipulate text files



This example demonstrates how to create a file and modify its contents using simple commands.



\--------------------------------------------------

 Creating a File with echo

\--------------------------------------------------



Command:



echo 'Hello' > hey.txt



Example terminal output:



┌──(kali㉿kali)-\[\~]

└─$ echo 'Hello' > hey.txt



This command created a file called:



hey.txt



The file was created in the current working directory:



\~



Which represents the user's home directory.



\--------------------------------------------------

 Viewing the File

\--------------------------------------------------



Command:



cat hey.txt



Example:



┌──(kali㉿kali)-\[\~]

└─$ cat hey.txt

Hello



The cat command prints the contents of the file to the terminal.



In this case, the file contains the text:



Hello



\--------------------------------------------------

 Overwriting File Content

\--------------------------------------------------



Command:



echo 'Hello again' > hey.txt



Example:



┌──(kali㉿kali)-\[\~]

└─$ echo 'Hello again' > hey.txt



When the file is viewed again:



cat hey.txt



Output:



Hello again



 Important behavior:



The single ">" symbol overwrites the file contents.



The original text ("Hello") is replaced with the new content.



\--------------------------------------------------

 Appending Content to a File

\--------------------------------------------------



Command:



echo 'Hello again again' >> hey.txt



Example:



┌──(kali㉿kali)-\[\~]

└─$ echo 'Hello again again' >> hey.txt



Now when we display the file again:



cat hey.txt



Output:



Hello again

Hello again again



\--------------------------------------------------

 Understanding Redirection Symbols

\--------------------------------------------------



Linux uses special symbols called redirection operators.



These operators control how command output is written to files.



\--------------------------------------------------



>



Single ">" redirects output and overwrites the file.



Example:



echo "text" > file.txt



This replaces the file contents.



\--------------------------------------------------



>>



Double ">>" appends output to the file.



Example:



echo "text" >> file.txt



This adds the text to a new line without removing the existing content.



\--------------------------------------------------

 Key Takeaways

\--------------------------------------------------



✔ echo can be used to create and write to files.



✔ cat is used to display file contents in the terminal.



✔ The ">" symbol overwrites file contents.



✔ The ">>" symbol appends new content to the file.



✔ Redirection operators are extremely useful for scripting and automation.



\--------------------------------------------------

 Security Relevance

\--------------------------------------------------



File redirection is commonly used during penetration testing to:



• save command output

• build scripts

• log enumeration results

• store extracted data



Understanding how output redirection works allows security professionals to efficiently capture and organize information during assessments.
