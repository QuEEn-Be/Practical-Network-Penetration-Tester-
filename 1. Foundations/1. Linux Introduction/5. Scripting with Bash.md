==================================================
 SCRIPTING WITH BASH – IP SWEEP AUTOMATION
==================================================

 Overview
--------------------------------------------------

One of the biggest advantages of Bash scripting is the ability to automate repetitive tasks.

In this example, a ping command is used to identify live hosts, then the output is narrowed down until only the IP address remains.

That logic is then turned into a Bash script that performs a simple IP sweep across a subnet.

This workflow demonstrates:

• output redirection
• filtering with grep
• selecting fields with cut
• removing unwanted characters with tr
• looping through IP ranges with Bash
• automating scans with nmap

--------------------------------------------------
 Saving Ping Output to a File
--------------------------------------------------

Command:

ping 192.168.56.1 -c 1 > ip.txt

This sends one ICMP request to the target and saves the output into a file called:

ip.txt

--------------------------------------------------

Example:

┌──(kali㉿kali)-[~]
└─$ ping 192.168.56.1 -c 1 > ip.txt

--------------------------------------------------
 Viewing the Output File
--------------------------------------------------

Command:

cat ip.txt

Example output:

PING 192.168.56.1 (192.168.56.1) 56(84) bytes of data.
64 bytes from 192.168.56.1: icmp_seq=1 ttl=127 time=0.688 ms

--- 192.168.56.1 ping statistics ---
1 packets transmitted, 1 received, 0% packet loss, time 0ms
rtt min/avg/max/mdev = 0.688/0.688/0.688/0.000 ms

This contains more information than needed, so the next step is to extract only the useful part.

--------------------------------------------------
 Filtering the Output with grep
--------------------------------------------------

Command:

cat ip.txt | grep "64 bytes"

Output:

64 bytes from 192.168.56.1: icmp_seq=1 ttl=127 time=0.688 ms

This narrows the output down to the line that confirms a reply was received.

--------------------------------------------------
 Extracting the IP Address with cut
--------------------------------------------------

Command:

cat ip.txt | grep "64 bytes" | cut -d " " -f 4

Output:

192.168.56.1:

Explanation:

• cut separates the line into fields
• -d " " uses a space as the delimiter
• -f 4 selects the fourth field

This gives the IP address, but it still includes a trailing colon.

--------------------------------------------------
 Cleaning the Output with tr
--------------------------------------------------

Command:

cat ip.txt | grep "64 bytes" | cut -d " " -f 4 | tr -d ":"

Output:

192.168.56.1

Now the output has been narrowed down to only the IP address.

--------------------------------------------------
 Turning It into a Bash Script
--------------------------------------------------

A Bash script can automate this same logic across a range of hosts.

To create the script:

sudo su
mousepad ipsweep.sh

Example script contents:

#!/bin/bash

for ip in `seq 1 25`; do
ping -c 1 $1.$ip | grep "64 bytes" | cut -d " " -f 4 | tr -d ":" &
done

--------------------------------------------------
 Script Explanation
--------------------------------------------------

#!/bin/bash
Tells the system to run the script using Bash.

for ip in `seq 1 25`
Loops through the numbers 1 to 25.

$1.$ip
Uses the first script argument as the subnet and appends the host number.

Example:
If the script is run with:

./ipsweep.sh 192.168.1

The loop checks:

192.168.1.1
192.168.1.2
192.168.1.3
...
192.168.1.25

&
Runs each ping in the background for faster execution.

--------------------------------------------------
 Making the Script Executable
--------------------------------------------------

Command:

chmod +x ipsweep.sh

This adds execute permissions so the script can be run directly.

--------------------------------------------------
 Running the Script
--------------------------------------------------

Example:

./ipsweep.sh 192.168.1

Example output:

192.168.1.4
192.168.1.1
192.168.1.3
192.168.1.7
192.168.1.5
192.168.1.8

These are the live hosts discovered during the sweep.

--------------------------------------------------
 Saving the Results to a File
--------------------------------------------------

Command:

./ipsweep.sh 192.168.1 > iplist.txt

This stores the discovered live IP addresses in a file.

To view the saved results:

cat iplist.txt

Example output:

192.168.1.1
192.168.1.4
192.168.1.7
192.168.1.5
192.168.1.3
192.168.1.8
192.168.1.14

--------------------------------------------------
 Automating Nmap Scans from the IP List
--------------------------------------------------

Once the live host list is saved, the file can be used to automatically scan each discovered host.

Command:

for ip in $(cat iplist.txt); do nmap -sS -p 80 -T4 $ip & done

This loop:

• reads each IP from iplist.txt
• runs an Nmap SYN scan
• scans port 80
• uses aggressive timing with -T4
• runs scans in the background

Example output:

--------------------------------------------------
 Improved Version of the Script
--------------------------------------------------

A better version of the script includes basic input validation.

Example:

#!/bin/bash

if [ "$1" == "" ]
then
    echo "You forgot an IP address!"
    echo "Syntax: ./ipsweep.sh 10.1.31"
else
    for ip in `seq 1 25`; do
        ping -c 1 $1.$ip | grep "64 bytes" | cut -d " " -f 4 | tr -d ":" &
    done
fi

--------------------------------------------------
 Key Takeaways
--------------------------------------------------

✔ Bash can be used to automate host discovery.

✔ grep helps filter useful output from a command.

✔ cut extracts specific fields from command output.

✔ tr removes unwanted characters from strings.

✔ Output can be redirected into files for reuse in later commands.

✔ Bash loops make it easy to repeat actions across a subnet.

--------------------------------------------------
 Security Relevance

• reduce repetitive manual work

Small scripts like this are often the foundation of larger offensive workflows.
