🌐 COMMON NETWORK COMMANDS

==================================================



📌 Overview

\--------------------------------------------------



Network commands are foundational in Linux administration and penetration testing.



These commands help identify:



• local IP addressing

• network neighbors

• routing paths

• host availability

• open ports and listening services



Understanding these tools is important because they are often the first commands used during enumeration and troubleshooting.



\--------------------------------------------------

🖧 Viewing IP Address Information

\--------------------------------------------------



Two common commands used to display IP configuration are:



ip a



and



ifconfig



These commands show:



• interface names

• IPv4 and IPv6 addresses

• MAC addresses

• interface state

• broadcast information



\--------------------------------------------------



Example:



ip a



This command displays all interfaces and their assigned addresses.



Example output may include:



• lo = loopback interface

• eth0 = network interface

• inet = IPv4 address

• inet6 = IPv6 address



\--------------------------------------------------



Example:



ifconfig



This is another common command used to display interface information.



It provides details such as:



• IP address

• netmask

• broadcast address

• RX packets

• TX packets

• interface errors



📌 Note:

The ip command is the more modern replacement for ifconfig, but both are still commonly seen in labs and walkthroughs.



\--------------------------------------------------

👥 Viewing Neighbor Information

\--------------------------------------------------



To see neighboring systems or devices on the local network, you can use:



arp -a



or



ip n



These commands help identify nearby hosts that your system has communicated with.



Common uses:



• identify local devices

• view ARP cache entries

• support local network enumeration



\--------------------------------------------------

🛣️ Viewing the Routing Table

\--------------------------------------------------



To display the routing table, use:



ip r



Example output:



default via 10.0.2.2 dev eth0 proto dhcp src 10.0.2.15 metric 100

10.0.2.0/24 dev eth0 proto kernel scope link src 10.0.2.15 metric 100



This tells you:



• the default gateway

• which interface is being used

• the local subnet

• how traffic leaves the machine



\--------------------------------------------------



Another command that shows routing information is:



route



Example output:



Kernel IP routing table

Destination     Gateway         Genmask         Flags Metric Ref    Use Iface

default         10.0.2.2        0.0.0.0         UG    100    0        0 eth0

10.0.2.0        0.0.0.0         255.255.255.0   U     100    0        0 eth0



📌 Why this matters:

The routing table shows how the machine reaches local and external networks. This is useful when troubleshooting connectivity or mapping network paths during enumeration.



\--------------------------------------------------

📡 Testing Host Availability with Ping

\--------------------------------------------------



The ping command checks whether another host is reachable on the network using ICMP.



Example:



ping 192.168.56.1



Example output:



PING 192.168.56.1 (192.168.56.1) 56(84) bytes of data.

64 bytes from 192.168.56.1: icmp\_seq=1 ttl=127 time=1.39 ms

64 bytes from 192.168.56.1: icmp\_seq=2 ttl=127 time=1.48 ms

64 bytes from 192.168.56.1: icmp\_seq=3 ttl=127 time=1.39 ms

64 bytes from 192.168.56.1: icmp\_seq=4 ttl=127 time=1.35 ms

64 bytes from 192.168.56.1: icmp\_seq=5 ttl=127 time=1.39 ms

64 bytes from 192.168.56.1: icmp\_seq=6 ttl=127 time=1.74 ms

64 bytes from 192.168.56.1: icmp\_seq=7 ttl=127 time=1.38 ms



\--- 192.168.56.1 ping statistics ---

7 packets transmitted, 7 received, 0% packet loss, time 6011ms

rtt min/avg/max/mdev = 1.352/1.446/1.742/0.125 ms



\--------------------------------------------------



📌 Important Note:

Ping relies on ICMP. Some systems block ICMP, so a lack of ping response does NOT always mean the system is offline.



\--------------------------------------------------

🎯 Limiting Ping Output

\--------------------------------------------------



To send only one ping request, use:



ping 192.168.56.1 -c 1



Example output:



PING 192.168.56.1 (192.168.56.1) 56(84) bytes of data.

64 bytes from 192.168.56.1: icmp\_seq=1 ttl=127 time=0.781 ms



\--- 192.168.56.1 ping statistics ---

1 packets transmitted, 1 received, 0% packet loss, time 0ms

rtt min/avg/max/mdev = 0.781/0.781/0.781/0.000 ms



This is helpful when you want a quick confirmation without continuous output.



\--------------------------------------------------

🚪 Viewing Open Ports and Connections

\--------------------------------------------------



Another useful command is:



netstat



This command helps display:



• open ports

• listening services

• active network connections



📌 Why this matters:

During penetration testing and system analysis, netstat can help identify what services are exposed on a machine.



Example use cases:



• identify open listening ports

• determine what services are running

• verify whether a process is accepting network connections



\--------------------------------------------------

📚 Key Takeaways

\--------------------------------------------------



✔ ip a and ifconfig are used to display interface and IP information.



✔ arp -a and ip n help identify nearby hosts and ARP neighbor entries.



✔ ip r and route display the system routing table.



✔ ping checks host reachability, but blocked ICMP can affect results.



✔ netstat can help identify open ports and active connections.



\--------------------------------------------------

🛡️ Security Relevance

\--------------------------------------------------



These commands are frequently used during the early stages of penetration testing to:



• identify network configuration

• verify connectivity

• enumerate neighboring devices

• inspect routing paths

• identify exposed services



Mastering these commands provides a strong foundation for more advanced network enumeration.
